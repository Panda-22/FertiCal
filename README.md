# FertiCal

FertiCal 是一个用于营养液配方核算和反向生成施肥方案的前端工具。当前页面名称为「丰码有料 UniMix」，核心目标是把原水背景、A/B 桶母液配方、目标元素浓度放在同一张工作台中，快速判断最终工作液是否接近目标，并评估母液中的析出风险。

当前版本暂未启用配方库。原水数据目前支持上传报告或手动输入。

## 当前功能

- 功能 01：计算工作液浓度
  - 上传水质检测报告，或直接手动修改原水背景值
  - 导入目标文件与实际配方文件，自动填入目标值和 A/B 桶用量
  - 手动配置 A 桶、B 桶肥料、母液体积和稀释倍数
  - 实时计算理论工作液浓度、目标偏差、EC、pH 和 A/B 桶析出风险
  - 支持导出理论灌溉液计算结果

- 功能 02：设计施肥配方
  - 上传或手动录入原水背景
  - 选择目标预设，或手动录入目标元素浓度
  - 自动生成 3 套可选配方建议：
    - 目标贴合优先
    - 低氯低硫优先
    - 原料成本最低
  - 点击任一方案后，方案会写入 A/B 桶并计算最终工作液浓度，另外两套方案仍会保留，方便继续对比
  - 选择 AB 肥配制量后，可将已选方案导出为 Excel 或 PDF

## 主要计算内容

- 原水背景：识别并记录 `N`、`NO3-N`、`NH4-N`、`P`、`K`、`Ca`、`Mg`、`S`、`Cl`、微量元素、`EC`、`pH`、`HCO3-` 等指标
- A/B 桶贡献：根据肥料元素百分比、用量、母液体积和稀释倍数，换算为工作液中的元素贡献
- 理论工作液：原水背景 + A 桶贡献 + B 桶贡献
- 目标偏差：计算理论值与目标值的偏差，偏差较大时在页面中提示
- pH 估算：结合原水碱度、硝酸、磷酸、磷酸二氢钾等酸性来源进行估算
- 析出风险：估算 A/B 桶中 `CaSO4·2H2O` 和 `Ca3(PO4)2` 的离子积风险；IP 与 Ksp 用 `mol/L` 浓度近似活度进行比较，单位分别为 `(mol/L)^2` 和 `(mol/L)^5`
- 方案评分：在接近目标的基础上，对 `Cl` 和 `S` 做低含量偏好，并可使用测试单价估算原料成本

## 支持导入的文件

- 水质报告：`.xlsx`、`.xls`、`.csv`
- 目标文件：`.pdf`、`.xlsx`、`.xls`、`.csv`、`.txt`、图片
- 配方文件：`.xlsx`、`.xls`、`.csv`、`.txt`、`.pdf`、图片

图片和 PDF 的识别能力取决于浏览器环境中的可用解析能力；表格类文件识别更稳定。

## 使用方法

### 直接打开

在浏览器中打开 [index.html](./index.html) 即可使用。

### 本地运行

首次使用前安装依赖：

```bash
npm install
```

然后用任意静态文件服务打开项目目录即可。也可以直接打开 `index.html`，但静态服务在文件导入和调试时更稳定。

### GitHub Pages 试发布

项目已包含 `.github/workflows/pages.yml`，推送到 `main` 后会自动发布前端文件：

1. 在 GitHub 仓库进入 `Settings` → `Pages`。
2. `Source` 选择 `GitHub Actions`。
3. 推送代码到 `main`，等待 `Deploy frontend to GitHub Pages` 工作流完成。
4. 打开 Pages 给出的地址，通常是 `https://panda-22.github.io/FertiCal/`。

GitHub Pages 只发布前端文件，手动录入、Excel/CSV 导入、计算和配方建议可用于手机试用；PDF/图片 OCR、依赖后端的上传解析接口暂不包含在这个前端发布版本里。

## 页面流程

1. 选择功能模式：计算已有配方，或设计新配方。
2. 在“水质背景”中导入报告，或直接输入原水元素值。
3. 如果是功能 01，录入或导入 A/B 桶配方。
4. 如果是功能 02，录入目标元素浓度，等待系统生成 3 套建议。
5. 点击某一套建议后，查看最终工作液浓度、偏差和析出风险。
6. 确认方案后，选择 `100 L` 或 `1000 L` 配制量，并导出 Excel 或 PDF。

## 依赖

- [`xlsx`](https://www.npmjs.com/package/xlsx)：解析和导出 Excel 文件

## 已知边界

- pH 和析出风险为配方设计阶段的估算结果，适合用于方案比较和风险预警，实际生产仍建议结合检测与小样验证。
- 不同实验室报告字段命名不完全一致，自动识别后仍建议人工检查关键原水指标。
